STEP 1: UPDATE DATABASE TO POSTGRESQL
═══════════════════════════════════════════════════════════════

Open: prisma/schema.prisma

Change line 13 from:
  provider = "sqlite"

To:
  provider = "postgresql"

Save the file.


STEP 2: GENERATE SESSION SECRET
═══════════════════════════════════════════════════════════════

Run this in PowerShell:
  -join ((65..90) + (97..122) + (48..57) | Get-Random -Count 32 | % {[char]$_})

Copy the output - you'll need it in Step 5!

Example output: k3Jd9mN2pQ8rT5vX7wZ1aB4cE6fG9hI2


STEP 3: PUSH TO GITHUB
═══════════════════════════════════════════════════════════════

A. Initialize git (if not done):
   git init
   git add .
   git commit -m "Initial commit"

B. Create repo on GitHub:
   - Go to github.com
   - Click "+" → "New repository"
   - Name it "kappa-tracker"
   - Don't initialize with anything
   - Click "Create repository"

C. Push your code:
   git remote add origin https://github.com/YOUR_USERNAME/kappa-tracker.git
   git branch -M main
   git push -u origin main


STEP 4: DEPLOY TO RAILWAY
═══════════════════════════════════════════════════════════════

A. Go to: https://railway.app

B. Click "Start a New Project"

C. Sign up with GitHub (fastest)

D. Click "Deploy from GitHub repo"

E. Select "kappa-tracker" repository

F. Railway will start building...


STEP 5: ADD DATABASE
═══════════════════════════════════════════════════════════════

A. In your Railway project, click "New"

B. Select "Database"

C. Choose "Add PostgreSQL"

D. Done! DATABASE_URL is automatically set


STEP 6: ADD ENVIRONMENT VARIABLES
═══════════════════════════════════════════════════════════════

A. Click on your web service (not the database)

B. Click "Variables" tab

C. Click "New Variable"

D. Add these two variables:

   Variable 1:
   - Key: SESSION_SECRET
   - Value: (paste the secret from Step 2)

   Variable 2:
   - Key: NODE_ENV
   - Value: production

E. Click "Add" for each


STEP 7: WAIT FOR DEPLOYMENT
═══════════════════════════════════════════════════════════════

A. Railway will automatically redeploy (2-3 minutes)

B. Watch the "Deployments" tab for progress

C. When it says "Success" → you're live!


STEP 8: GET YOUR URL
═══════════════════════════════════════════════════════════════

A. Click "Settings" tab

B. Under "Domains" click "Generate Domain"

C. Railway will create a URL like:
   kappa-tracker-production-xxxx.up.railway.app

D. Click the URL to visit your site!


STEP 9: TEST IT!
═══════════════════════════════════════════════════════════════

A. Visit your Railway URL

B. Click "Create Account"

C. Register with test credentials

D. Track some quests

E. Check the rankings page

F. Success! 🎉


STEP 10 (OPTIONAL): ADD CUSTOM DOMAIN
═══════════════════════════════════════════════════════════════

If you have a domain:

A. In Railway: Settings → "Custom Domain"

B. Enter your domain: tracker.yourdomain.com

C. Railway will show you DNS records to add

D. Go to your domain registrar (GoDaddy, Namecheap, etc.)

E. Add CNAME record:
   - Type: CNAME
   - Name: tracker
   - Value: (from Railway)
   - TTL: 3600

F. Wait 10-30 minutes for DNS to update

G. Visit your custom domain - it works!


TROUBLESHOOTING
═══════════════════════════════════════════════════════════════

❌ "Build failed"
   → Check Railway logs
   → Make sure you changed sqlite to postgresql
   → Push the change to GitHub

❌ "Application failed to respond"
   → Check that SESSION_SECRET is set
   → Check that DATABASE_URL exists (should be automatic)
   → View logs in Railway dashboard

❌ "Can't login after deployment"
   → Register a new account (fresh database!)
   → Clear browser cookies
   → Try incognito mode

❌ GitHub push fails
   → Make sure you created the repo on GitHub first
   → Check the remote URL is correct
